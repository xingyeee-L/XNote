# 德语锚点笔记系统 (XNote) - 完整产品需求文档 (PRD) v1.1

## 1. 项目概述与核心目标

### 1.1 项目定义
XNote 是一款面向德语学习者的、以"课文场景"为核心锚点的笔记整理与复习系统。它摒弃了孤立背单词的传统模式，强制要求所有词汇和语法卡片必须依附于某篇具体课文的上下文。

### 1.2 核心业务指标 (KPI for V1)
- 纯净盲读体验：阅读时默认隐藏所有翻译和语法，保障沉浸感
- 场景高复用性：在不同课文中出现的同一个词，映射到同一个知识卡片，实现多处语境回溯
- 主动召回机制：基于真实课文挖空的闪卡复习系统，抗遗忘
- 安全只读分享：网站默认公开给外部访客，写操作（增删改）严格受密码保护

## 2. 页面结构与路由规划

| 路由路径 | 对应组件/页面 | 访问权限 | 核心功能 |
|----------|--------------|----------|----------|
| `/` | 首页 (Home Page) | 公开只读 | 展示历史课文列表、学习进度概览、进入复习面板入口 |
| `/reader/[textId]` | 阅读器页面 (Reader View) | 公开只读（登录后桌面端可编辑） | 系统核心交互区。展示课文，点击单词唤出边栏，展示语境回溯 |
| `/review` | 复习面板 (Review Panel) | 公开只读 | 独立复习闪卡页面，支持标签过滤、卡片翻转与挖空训练 |
| `/admin` | 管理登录页 (Admin Portal) | 公开 | 输入密码，获取管理员写权限 HTTP-only Cookie |

## 3. 核心功能详细规格

### 功能 3.1：原文"对象化"解析与渲染引擎 (Word-as-Object Engine)

**数据存储逻辑：**
课文原文存储为"Token 对象"数组的 JSONB，而非单一 string。

**Token 对象结构：**
```typescript
type Token = {
  t: string;              // 单词文本或符号，如 "bestellen", " ", "."
  note_id: number | null; // 绑定的知识卡片 ID。null 代表普通文本
};
```

**渲染交互规则：**
- `note_id === null`：直接渲染为普通文本
- `note_id !== null`：
  - 默认状态：灰色虚线下划线（Dotted Underline）
  - Hover：下划线变蓝，光标变为指针
  - Click：右侧滑出该 note_id 对应的知识卡片

### 功能 3.2：隐形侧边栏与语境回溯 (Collapsible Side-Margin)

**布局与动画：**
- Desktop：侧边栏 (宽 350-400px) 默认从右侧滑出，课文区域自动缩进
- Mobile：底部抽屉（Bottom Drawer）向上弹出，手势滑动关闭
- 动画时长 300ms，ease-in-out 缓动

**卡片展示内容：**
- 头部：核心词汇/短语 + 标签组
- 中部：释义与语法剖析，支持 Markdown 渲染
- 底部（语境回溯区域）：反向查询其他包含此 note_id 的课文列表

**语境回溯 Modal：**
- 点击列表项后，屏幕中央弹出模态预览窗
- 异步拉取课文，自动定位并高亮该词
- **只读预览**，不可编辑
- 右上角"关闭"按钮，关闭后留在当前阅读页

**移动端适配：**
- 主要使用阅读模式，不开放编辑功能
- 侧边栏以 Bottom Drawer 形式呈现

### 功能 3.3：轻量级闪卡主动召回复习 (Active Recall Panel)

复习页面位于 `/review`，采用无干扰专注模式。

**卡片双面渲染逻辑：**

正面（测试面）：
- 获取卡片绑定的真实课文原句
- 将该词用 `______` 或高亮遮挡块代替，逼迫用户上下文召回

背面（释义面）：
- 点击"揭晓答案"按钮，翻转或移除模糊
- 展示完整释义、变形及标签

**队列算法 (V1)：**
1. 进入页面，从 API 获取所有复习卡片列表
2. Shuffle 洗牌打乱顺序
3. 难度标记：
   - **Next**：卡片移出队列，加载下一张
   - **Hard**：卡片重新插入队列末尾，**最多循环 3 次**
4. 3 次后自动标记为"已掌握"，不再进入复习队列

### 功能 3.4：管理员"就地编辑"工作台 (In-situ Editor)

**身份认证：**
- 访问 `/admin` 输入密码
- 验证成功：服务端下发 HTTP-only Cookie（安全存储，永不硬编码）
- 密码从环境变量 `ADMIN_PASSWORD` 读取，建议使用 `openssl rand -base64 24` 生成

**编辑权限：**
- 登录状态：桌面端界面顶部显示"管理员模式已开启"提示条
- 移动端不开放编辑功能

**绑定已有卡片：**
1. 选中未高亮文本，弹出悬浮按钮 [绑定/新建卡片]
2. 右侧弹出编辑抽屉，提供模糊搜索框
3. 输入字母搜索数据库已存在 notes
4. 选中记录并确认，更新 tokens 数组中的 note_id

**新建卡片：**
1. 搜索无结果时点击"创建新卡片"
2. 抽屉内填写：词汇名、变形、释义（Markdown）、标签
3. 保存后自动创建 note 记录并绑定到对应 Token

## 4. UI/UX 视觉设计规范

**色调风格：**
- 背景色：暖白/米色（如 `slate-50` 或 `amber-50/10`），降低长时间阅读疲劳
- 主色调：深灰/炭黑（如 `zinc-800`），保证文字可读性
- 高亮色：浅蓝或莫兰迪灰（如 `sky-100` 或 `sky-900/20`），仅用于锚点提示

**字体规范：**
- 德语原文：衬线体（Serif，如 Georgia），模拟实体德语小说阅读感
- 翻译与界面：无衬线体（Sans-serif，如 Inter）

**响应式要求：**
- 强制适配移动端
- 手机端复习时卡片占满屏幕，按钮尺寸 ≥ 44px × 44px 防误触
- 移动端以阅读为主，不开放编辑

## 5. 安全性与接口防御规则

**接口只读限制：**
- GET 请求（获取课文、卡片列表）：完全公开，无需身份校验
- POST / PUT / DELETE 请求（新建卡片、更新 tokens、删除笔记）：必须携带合法 Cookie，否则返回 401

**Cookie 安全策略：**
- HTTP-only：防止 JavaScript 读取
- Secure：生产环境启用
- SameSite=Lax：跨站请求携带 Cookie
